What Trueform does

Four ways to start a design.

Trueform meets you wherever the idea begins — a file, a sentence, a sketch, or a photograph of something that snapped.

1. Upload a CAD file

Drop in STEP or STL. Trueform parses the geometry and extracts the design features it needs to reason about.

2. Describe it in words

Tell the agent what the part has to do. It builds the CAD with you, iteratively refining geometry, fits, and constraints.

3. Sketch it

Draw on paper or a tablet, snap it. Trueform interprets the lines and converts them into parametric CAD.

4. Photograph the broken part

One image of a snapped component is enough. Trueform reconstructs the geometry and proposes a stronger replacement.

The moat

Bound by the rules of mechanics.

The language model is a router, not an oracle. It is forbidden from generating numerical answers.

  • The LLM only routes work to validated physics solvers. It never invents a number.

  • Every output is computed by a deterministic solver and cites the source equation, page, and standard — Bruhn, Niu, ESDU, Lekhnitskii, ASME Y14.5.

  • Every recommendation passes a physics-and-manufacturability constraint check — yield, buckling, fatigue, printability, tolerance feasibility — before emission.

  • If no validated solver can answer a question, the agent says so and stops.

What it emits

Drawing pack

An ASME Y14.5-compliant, GD&T-annotated drawing set, ready for the shop.

Analysis report

A written report citing the relevant Bruhn, Niu, ESDU, and standard passages line by line.

Manufacturing code

NC / G-code for machining, sliced output for 3D printing — the part, ready to make.

12-month roadmap

From intake to manufactured part, in four quarters.

  1. M1–3

    Intake layer

    STEP/STL parsing, photo-to-CAD reconstruction, conversational CAD generation, feature extraction. Open-sourcing the underlying classical sizing tools.

  2. M4–6

    Sizing engine wired in

    LLM dispatcher, with constraint enforcement happening before any output leaves the agent.

  3. M7–9

    Inverse parametric design

    Pareto-front optimisation across mass, cost, and life. ASME Y14.5 GD&T generation.

  4. M10–12

    Manufacturing handoff & v1.0

    Public v1.0 with four worked examples — a kitchen-chair clip rebuilt from a photo, a stress bracket, a fatigue lug, and a pressure-vessel section.

Proof of capability

Built on tools already shipped.

Trueform isn't starting from zero. The numerical machinery underneath has been written, validated, and in some cases used commercially.

BJSFM

An already-shipped Lekhnitskii anisotropic stress-around-hole solver. MIT-licensed.

github.com/allamaprabhuani/bjsfm →

Differentiable phase-field fracture

A GPU-native, end-to-end-differentiable phase-field fracture solver in PyTorch. Roughly 10× faster than reference codes; validated against Akantu, FEniCSx, COMSOL, and PhaseFieldX.

Classical structural-analysis library

Lug strength, Cozzone plastic-bending, fastener load-transfer, ABD matrices, column buckling — built during the founder's MTech and shipped commercially through Aeroknacks India Ltd.
